tiledb.cloud.bioimg.exportation.export¶

tiledb.cloud.bioimg.exportation.export(source: Sequence[str] | str, output: str, config: Mapping[str, Any], *args: Any, taskgraph_name: str | None = None, num_batches: int | None = None, resources: Mapping[str, Any] | None = None, compute: bool = True, namespace: str | None = None, **kwargs) → DAG¶

The function exports microscopy images from TileDB arrays

Parameters:

  • source – uri / iterable of uris of input files

  • output – output dir for the exported tiledb arrays

  • config – dict configuration to pass on tiledb.VFS

  • taskgraph_name – Optional name for taskgraph, defaults to None

  • num_batches – Number of graph nodes to spawn. Performs it sequentially if default, defaults to 1

  • threads – Number of threads for node side multiprocessing, defaults to 8

  • resources – configuration for node specs e.g. {“cpu”: “8”, “memory”: “4Gi”}, defaults to None

  • compute – When True the DAG returned will be computed inside the function

otherwise DAG will only be returned. :param namespace: The namespace where the DAG will run
