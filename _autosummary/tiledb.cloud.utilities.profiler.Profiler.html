tiledb.cloud.utilities.profiler.Profiler¶

class tiledb.cloud.utilities.profiler.Profiler(*, array_uri: str | None = None, group_uri: str | None = None, group_member: str | None = None, id: str | None = None, period_sec: int = 5, trace: bool = False)¶

Bases: object

A context manager–based profiler to log events and CPU and memory usage to a TileDB array.

If the trace parameter is True, CPU and memory usage will be logged to the array every period_sec seconds. This is useful for profiling jobs that are OOM killed.

Examples:

# Basic usage with Profiler(array_uri=”tiledb://array-uri…”):

# code to profile

# Write custom events with Profiler(group_uri=”tiledb://group-uri…”, group_member=”log”) as prof:

# code to profile

# write custom event prof.write(“my-op”, “my-data”, “my-extra-data”)

# more code to profile

__init__(*, array_uri: str | None = None, group_uri: str | None = None, group_member: str | None = None, id: str | None = None, period_sec: int = 5, trace: bool = False)¶

Create a profiler object which logs events to a TileDB array. The array can be specified by URI or by group URI and group member name.

Parameters:

  • array_uri – URI of the log array, defaults to None

  • group_uri – URI of the group containing the log array, defaults to None

  • group_member – group member name of the log array, defaults to None

  • id – profiler id, written to event id

  • period_sec – profiling period in seconds (0 = disabled), defaults to 5

  • trace – enable trace logging, defaults to False
