tiledb.cloud.vcf.ingestion.ingest_samples_dag¶

tiledb.cloud.vcf.ingestion.ingest_samples_dag(dataset_uri: str, *, config: Mapping[str, Any] | None = None, namespace: str | None = None, contigs: Sequence[str] | Contigs | None = Contigs.ALL, threads: int = 8, batch_size: int = 10, workers: int = 40, max_samples: int | None = None, resume: bool = True, ingest_resources: Mapping[str, str] | None = None, verbose: bool = False, create_index: bool = True, trace_id: str | None = None, batch_mode: bool = True, access_credentials_name: str | None = None, compute: bool = True, consolidate_stats: bool = False) → Tuple[DAG | None, Sequence[str]]¶

Create a DAG to ingest samples into the dataset.

Parameters:

  • dataset_uri – dataset URI

  • config – config dictionary, defaults to None

  • namespace – TileDB-Cloud namespace, defaults to None

  • contigs – contig mode (Contigs.ALL | Contigs.CHROMOSOMES | Contigs.OTHER | Contigs.ALL_DISABLE_MERGE) or list of contigs to ingest, defaults to Contigs.ALL

  • threads – number of threads to use per ingestion task, defaults to VCF_THREADS

  • batch_size – sample batch size, defaults to VCF_BATCH_SIZE

  • workers – maximum number of parallel workers, defaults to VCF_WORKERS

  • max_samples – maximum number of samples to ingest, defaults to None (no limit)

  • resume – enable resume ingestion mode, defaults to True

  • ingest_resources – manual override for ingest UDF resources, defaults to None

  • verbose – verbose logging, defaults to False

  • create_index – force creation of a local index file, defaults to True

  • trace_id – trace ID for logging, defaults to None

  • batch_mode – run all DAGs in batch mode, defaults to True

  • access_credentials_name – name of role in TileDB Cloud to use in tasks

  • compute – when True the DAG will be computed before it is returned, defaults to True

  • consolidate_stats – consolidate the stats arrays, defaults to False

Returns:

sample ingestion DAG and list of sample URIs ingested
