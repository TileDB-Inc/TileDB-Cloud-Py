tiledb.cloud.taskgraphs.builder.TaskGraphBuilder¶

class tiledb.cloud.taskgraphs.builder.TaskGraphBuilder(name: str | None = None)¶

Bases: object

The thing you use to build a task graph.

This class only builds task graphs. The graphs it builds are static and only represent the steps to run (the recipe). The actual execution will be later performed by the executor.

array_read(uri: str | Node[_T], *, raw_ranges: Tuple[Any, ...] | List[Any] | Node[_T] | Tuple[Any | Node[_T], ...] | List[Any | Node[_T]] | None = None, buffers: Tuple[str, ...] | List[str] | Node[_T] | Tuple[str | Node[_T], ...] | List[str | Node[_T]] | None = None, layout: Layout | str | None = None, name: str | None = None) → Node[Dict[str, ndarray]]¶

Creates a Node that will read data from a TileDB array.

This Node is not executed immediately; instead, it is used in the same way as the array input to an Array UDF works: when an actual UDF is executed, the array is queried server-side and is passed as a parameter to the user code.

Parameters:

  • uri – The URI to query against. This must be a tiledb:// URI. May be provided either as the URI itself, or as the output of an upstream node.

  • raw_ranges –

    The ranges to query against. This is called “raw” because we accept the format that is passed to the server:

    [
    [startDim1A, endDim1A, startDim1B, endDim1B, ...],
    [startDim2A, endDim2A, startDim2B, endDim2B, ...],
]

    This may also be provided as either a value or a Node output.

  • buffers – Optionally, the buffers to query against. May be either a raw value or the Node output.

  • name – An optional name for this Node.

Parameters:

  • name – The name of this input. Required, since it is used when executing to match the input to the Node.

  • default_value – An optional default value to use when executing. If not provided, the caller is required to set this input when running the task graph.

sql(query: str, init_commands: Iterable[str] = (), parameters: Tuple[_T, ...] | List[_T] | Node[_T] | Tuple[_T | Node[_T], ...] | List[_T | Node[_T]] = (), *, result_format: str = 'arrow', resource_class: str | None = None, download_results: bool | None = None, namespace: str | None = None, name: str | None = None) → Node¶

Creates a Node that executes an SQL query.

Parameters:

  • query – The query to execute. This must be a string, and cannot be the output of a previous node.

  • init_commands – A list of SQL commands to execute in the session before running query.

  • parameters – A sequence of objects to provide as parameters for the ? placeholders in the query. These may be provided either as values or as the output of earlier Nodes.

  • result_format – The format to provide results in. Either json or arrow.

  • resource_class – If specified, the container resource class that this UDF will be executed in.

  • download_results – If True, download results eagerly (i.e., immediately when the function returns). If False, download results lazily (i.e., only when you call .result() on an execution). If unset (the default), automatically choose whether to download results: eagerly if it’s a terminal node, or if it has a local dependent; lazily if it’s an internal node.

udf(func: str | Callable[[...], _T], args: Arguments = args(), *, result_format: str | None = 'python_pickle', include_source: bool = True, image_name: str | None = None, timeout: timedelta | int | None = None, resource_class: str | None = None, namespace: str | None = None, name: str | None = None, local: bool = False, download_results: bool | None = None) → Node[_T]¶

Creates a Node which executes a UDF.

Parameters:

  • func – The function to call; either a Python callable or a registered UDF name.

  • args – The arguments to pass to this function. These may contain values or Nodes.

  • result_format – The format to return results in.

  • include_source – True (the default) to include the function source in the request. This is useful for debugging and logging, but does not have any impact on the UDF’s execution. False to omit source.

  • image_name – If specified, will execute the UDF within the specified image rather than the default image for its language.

  • timeout – If specified, the length of time after which the UDF will be terminated on the server side. If specified as a number, a number of seconds. If zero or unset, the UDF will run until the server’s configured maximum. Unlike the timeout parameter to Future-like objects, this sets a limit on actual execution time, rather than just a limit on how long to wait.

  • resource_class – If specified, the container resource class that this UDF will be executed in.

  • namespace – If specified, the non-default namespace that the UDF will be executed under. This will also be the namespace used for reading any array nodes used in this UDF’s input.

  • local – If True, will attempt to run the UDF on the client machine. If this is not possible, the UDF will fail.

  • download_results – If True, download results eagerly (i.e., immediately when the function returns). If False, download results lazily (i.e., only when you call .result() on an execution). If unset (the default), automatically choose whether to download results: eagerly if it’s a terminal node, or if it has a local dependent; lazily if it’s an internal node.
