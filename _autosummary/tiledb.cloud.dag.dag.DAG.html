__init__(max_workers: int | None = None, use_processes: bool = False, done_callback: Callable[[DAG], None] | None = None, update_callback: Callable[[DAG], None] | None = None, namespace: str | None = None, name: str | None = None, mode: Mode = Mode.REALTIME, retry_strategy: RetryStrategy | None = None, workflow_retry_strategy: RetryStrategy | None = None, deadline: int | None = None)¶

DAG is a class for creating and managing direct acyclic graphs :param max_workers: how many workers should be used to execute the dag :param use_processes: if true will use processes instead of threads,

defaults to threads

Parameters:

  • done_callback – optional call back function to register for when dag is completed. Function will be passed reference to this dag

  • update_callback – optional call back function to register for when dag status is updated. Function will be passed reference to this dag

  • namespace – optional namespace to use for all tasks in DAG

  • name – A human-readable name used to identify this task graph in logs. Does not need to be unique.

  • mode – Mode the DAG is to run in, valid options are Mode.REALTIME, Mode.BATCH

  • retry_strategy – RetryStrategy to be applied on every node of the DAG.

  • workflow_retry_strategy – RetryStrategy to use to retry the entire DAG.

  • deadline – Duration in seconds relative to the workflow start time which the workflow is allowed to run before it gets terminated.
