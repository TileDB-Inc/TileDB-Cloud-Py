tiledb.cloud.taskgraphs.client_executor.impl.LocalExecutor¶

class tiledb.cloud.taskgraphs.client_executor.impl.LocalExecutor(graph: Dict[str, Any] | TaskGraphBuilder, namespace: str | None = None, *, api_client: Client | None = None, name: str | None = None, parallel_server_tasks: int = 10)¶

Bases: IClientExecutor

Coordinates the execution of a task graph locally.

__init__(graph: Dict[str, Any] | TaskGraphBuilder, namespace: str | None = None, *, api_client: Client | None = None, name: str | None = None, parallel_server_tasks: int = 10)¶

Sets up a local executor.

Parameters:

  • graph – The graph to execute, provided either as a JSON result or as the Builder object used to create the graph.

  • namespace – If provided, the namespace to run tasks in. If not provided, uses the default namespace.

  • api_client – An API client object, for using a client other than the default.

  • name – A name to give this execution of your task graph.

  • parallel_server_tasks – The maximum number of tasks that will be run on the server simultaneously.

add_update_callback(callback: Callable[[_ExSelf], None]) → None¶

Adds a callback that will be called when a node’s state changes.

Callbacks are delivered on a best-effort basis and may be asynchronous or batched depending upon the implementation. There is no guarantee as to what thread the callback may come in on.

cancel() → bool¶

If possible, cancels further execution of this graph.

Like futures.Future.cancel, this returns True if the graph could be cancelled, and False if not.

cancelled_nodes() → Tuple[_N, ...]¶

A snapshot of all nodes that have been cancelled.

This includes both nodes that were manually cancelled, and nodes that are not executed because their parents failed.
